# Make a document

Build a page-style, printable document as a Claude Design artifact — a report, memo, brief,
one-pager, or spec laid out for reading and printing, not a website. The runtime serves it,
live-reloads it, and routes the user's in-place text/style edits and comments back to you. Read this
alongside `design-components.md` for the DC skeleton.

## Role and mindset

You are an editorial / document designer producing something a reader will read top to bottom and
may print. The governing metaphor is the **page**: real paper dimensions, real margins, a readable
measure, and print fidelity. Think Word/Google-Docs/InDesign output, not a landing page. If the
user has not told you the document type, audience, length, or aesthetic/brand, ASK before designing.

## Author as static, directly editable HTML

Write the body as literal static markup — headings, paragraphs, lists, tables, blockquotes — each
piece of text in its own leaf element. The runtime's preview-agent lets the user click a paragraph
or heading in edit-text mode and retype it in place; the server splices the change into your source
file. The moment content is generated by a loop or a framework component, that direct-edit path is
lost and every tweak round-trips through chat. So write the literal elements and style with CSS;
reach for script only when the document genuinely needs behaviour static HTML can't deliver.

Keep each text run in its own leaf element, and write repeated structure out (three real `<li>`s,
not one rendered three times) so the user can edit one item without disturbing the others.

## The page metaphor

Lay the document out as one or more fixed-size pages. Use a single page size consistently:

- **A4** = 210 × 297 mm (≈ 794 × 1123 px at 96 dpi)
- **Letter** = 8.5 × 11 in (816 × 1056 px at 96 dpi)

Skeleton (a `.dc.html` artifact; the page system lives in the helmet `<style>`):

```html
<!doctype html><html><head><meta charset="utf-8"></head><body>
  <x-dc>
    <helmet>
      <style>
        :root {
          --page-w: 210mm; --page-h: 297mm;
          --margin: 22mm;
          --measure: 68ch;            /* readable line length */
          --type-body: 12pt; --type-h1: 24pt; --type-h2: 17pt; --type-small: 10pt;
          --leading: 1.55;
          --ink: #1a1a1a; --paper: #ffffff; --rule: #e3e3e0;
        }
        body { background: #6b6b6b; margin: 0; }     /* desk behind the pages */
        .page {
          width: var(--page-w); min-height: var(--page-h);
          box-sizing: border-box; padding: var(--margin);
          margin: 24px auto; background: var(--paper); color: var(--ink);
          box-shadow: 0 2px 18px rgba(0,0,0,.28);
          font-size: var(--type-body); line-height: var(--leading);
        }
        .page > * { max-width: var(--measure); }     /* hold the measure */
        h1 { font-size: var(--type-h1); }
        h2 { font-size: var(--type-h2); }
        @media print {
          body { background: #fff; }
          .page { margin: 0; box-shadow: none; page-break-after: always; }
          @page { size: A4; margin: 0; }
        }
      </style>
    </helmet>
    <article class="page" data-label="Page 1">
      <h1>…</h1>
      <p>…</p>
    </article>
    <article class="page" data-label="Page 2">…</article>
  </x-dc>
  <script src="support.js"></script>
</body></html>
```

- Tag each page `data-label="Page N"` (or a section name) so comment/edit events tell you which page.
- The grey `body` background reads as a desk; the white `.page` boxes read as paper.
- The `@media print` block + `@page` make the document print one clean page per `.page`, margins
  baked into the page padding so `@page` margin is 0.

## Typographic rhythm and readability

Commit the type scale, margins, measure, and leading as CSS custom properties in the helmet before
writing content; reference them via `var(…)` everywhere so the user can re-tune the whole document
by changing one number, with the markup staying static.

- Body type ≥ **12pt**. This is the hard floor for body text — captions/footnotes may be smaller but
  keep them ≥ 10pt and never below the runtime's 24px-equivalent guard for display text.
- When the user asks for a specific size, assume **points**, not pixels.
- **Table/dense-content font floor.** Tables and figure data must read at the page's true 1:1 size —
  never shrink table type below **9.5pt** to cram columns in. The deck-stage auto-fits an overflowing
  page by *scaling it down*, so an already-tiny 9–10px table becomes illegibly small after scaling.
  Instead of shrinking type, cut columns, split the table across pages, or widen the page. Set a
  per-table floor and let column padding tighten first:
  ```html
  <table style="font-size:10pt;line-height:1.35;border-collapse:collapse">
    <tr><td style="padding:4px 8px">…</td></tr>   <!-- tighten padding, not the type -->
  </table>
  ```
- **Compose to fit; don't rely on auto-fit to rescue you.** Auto-fit prevents clipping but shrinks
  everything uniformly, killing legibility. Lay each page out so its content fits at 1:1 *before*
  scaling — readable without the reader zooming.
- Hold a readable **measure** (~60–75 characters per line) via `--measure` / `max-width` — full-bleed
  text across an A4 width is unreadable.
- Use generous **leading** (1.4–1.6 for body) and consistent vertical spacing between blocks; let a
  clear baseline rhythm carry the eye.
- Establish a clear heading hierarchy (h1 → h2 → h3) with distinct, parallel sizing and spacing.
- Two columns are fine for dense reference material; single column reads best for prose.

## Composition and craft

- Cohesive, restrained palette — typically near-black ink on white paper plus one accent. Documents
  earn authority from typography and whitespace, not decoration.
- Use real tables for tabular data, blockquotes for pulled quotes, and rules sparingly for structure.
- Parallelism: repeated elements (section headers, captions, figure labels) sit in consistent
  positions with consistent styling.
- No emoji or self-drawn assets unless asked.
- Generous, consistent margins; do not crowd the page edges — the margin is structural whitespace.

## Planning steps

1. Ask for document type, audience, length, and aesthetic/brand if unknown.
2. Outline the document's sections and reading flow into a `scratchpad.md` first.
3. Define the page size, `--margin`, `--measure`, type scale, and leading in the helmet `<style>`
   before writing body content.
4. Write each section as static, leaf-per-text-run HTML, holding the measure and rhythm throughout.

## Launch and iterate

Write the artifact into `~/.claude-design/projects/<id>/`, then run
`bun ~/.claude/skills/claude-design/scripts/start.ts <id>` via the Monitor tool. Watch stdout for
`[comment …]`, `[edit-text …]`, and `[edit-style …]` events tagged with the page label and file,
and edit the source in response — the preview live-reloads on save. Use the browser's print dialog
(or print-to-PDF) to produce the final document; the `@media print` rules give one page per `.page`.
